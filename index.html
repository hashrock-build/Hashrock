How to Play

Mine ore seeded by the Solana blockhash, earn in-game coins, then redeem them for $HASHROCK.

1 · Enter

Click Play Now on the start page. A wallet is optional to start — you only need Phantom or Backpack (on Devnet) when you want to buy, upgrade, or redeem.

2 · Move & mine

  • WASD / arrow keys to walk around the village.
  • Stand next to blue crystal ore and press Space to mine. Moving cancels mining.
  • Ore spawns from the latest blockhash (≈1 per minute) and persists under a FIFO cap.
  • If several miners hit the same ore, the reward splits by damage contributed.

3 · Earn & grow

  • Mining pays in-game coins from the reward pool.
  • Marketplace — buy axe tiers (Wooden → Diamond) & color skins with $HASHROCK.
  • Upgrade — level each owned axe 1→10 for more mining throughput.
  • Repair — restore axe durability.

4 · Redeem

Open your Profile → Request Redeem to burn coins and release $HASHROCK 1:1 to your wallet.

Whitepaper

HASHROCK is built on one rule: the treasury can never owe more than it holds. Earning is redistribution, not emission.

The token

  • $HASHROCK — Solana SPL token, fixed 500,000 supply (1 token = 1 coin).
  • Mint authority is burned — no one can ever print more.
  • 50K (10%) seeds the Reward Pool as the play-to-earn budget.

Solvency invariants

  1. Coins are minted only when a $HASHROCK deposit is confirmed (1:1).
  2. Mining does not mint — rewards come from the Reward Pool, refilled by sinks.
  3. Total coins in circulation ≡ treasury, always 1:1 backed.
  4. Redeeming burns coins, then releases tokens. Evicted ore recycles to the pool.

Sinks & the fixed pie

Upgrades, repairs, and marketplace buys are paid on-chain in $HASHROCK — never in mined coins. Of every sink, 95% recycles to the pool (back to miners) and 5% is the creator cut. Daily ore & payout are fixed: upgrades shift your share of the pie, never grow emission.

Mining payout = a dynamic % of the current pool, so it self-balances — shrinking as the pool drains, growing as sinks refill it. It never hard-zeroes.

Docs

Built like infrastructure: a server-authoritative accounting system with a game on top.

Stack

  • Client — PixiJS v8 (top-down 2D), Vite + TypeScript.
  • Server — Colyseus (authoritative) + Postgres with an append-only ledger auditing the 1:1 invariant.
  • Chain — Solana ($HASHROCK SPL, treasury), on-chain for value only: transfers, sinks, deposit, redeem.

Security

  • Server validates movement, range & every reward — the client is never trusted for economic values.
  • Redeem requires a server-verified signature; purchases verify the on-chain payment before granting.
  • World layout & item stats stay off-chain (gas); only economic value lives on-chain.

Roadmap

  • M1 — Top-down render, camera, 4-dir player, blockhash ore + FIFO.
  • M2 — 112×112 village, terrain/forest/houses, culled ground, Pixel Crawler art, economy v2.
  • M3 — Authoritative Colyseus server + Postgres; on-chain deposit/redeem; marketplace (axe tiers + color skins) & per-tier axe leveling.
  • M4 — Live testnet, secondary player↔player marketplace, durability/repair depth, character preview polish.
  • M5 — Mainnet (treasury → multisig/program), themed mining biomes (cave / desert / forge / garden…).
